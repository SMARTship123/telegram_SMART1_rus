# Очередь постов

Планировщик (`.github/workflows/telegram-scheduler.yml`) публикует **один**
элемент из этой папки за запуск — 09:00 UTC ежедневно — в **порядке имён файлов**,
затем перемещает опубликованный файл в `queue/published/`.

То есть числовой префикс задаёт расписание: `0001-` уходит раньше `0002-`, и т.д.

## Формат элемента

Каждый элемент — `queue/NNNN-slug.md`: маленький заголовок, строка `---`, затем тело.

```
type: photo
image: images/moya-kartochka.png
---
<b>$330 тыс.</b> — подпись идёт здесь, HTML допустим, ≤1024 символов.

#вибромониторинг #СНО
```

- `type:` — `photo` или `text` (по умолчанию `text`).
- `image:` — путь к PNG в репозитории; **обязателен** при `type: photo`.
- Тело — HTML (`<b> <i> <u> <s> <a> <code> <pre>`), по `editorial/SYSTEM_PROMPT.md`.
  - Подпись к фото: **≤1024 символов** (отправка падает выше лимита).
  - Текстовое сообщение: ≤4096 символов; редакционный ориентир 800–1500 знаков.

## Добавить пост

1. Напиши контент по `editorial/SYSTEM_PROMPT.md` и `editorial/PUBLISHING_PREFERENCES.md`.
2. Для фото-поста собери карточку. Два генератора:
   - `scripts/make_card.py` — простая карточка-цифры («БОЛЬШОЕ=мелкое», до 3).
   - `scripts/make_visual.py` — богатые **карточки-графики**, по одной на `--kind`,
     чтобы канал не повторял один шаблон:
       - `spectrum` — столбцы частот вибрации, ключевые пики выделены (сигнатуры дефектов)
       - `trend`    — значение, растущее сквозь зоны ИСО A/B/C/D со временем
       - `timeline` — стадийное развитие дефекта с «окном экономии»
       - `delta`    — баланс «с наблюдением / без» с итоговой цифрой
       - `gauge`    — полукруглый индикатор тяжести/оценки (`good_high=True` — высокое=зелёное)
     Импортируемо: `import make_visual; make_visual.render(kind, out, title=…, **kw)`.
3. Создай `queue/<следующий-номер>-<slug>.md` с заголовком + телом (подпись ≤1024).
4. Закоммить и запушь. Пост подхватится на следующем запуске планировщика.

## Проверка без ожидания (и без публикации)

```
DRY_RUN=1 bash scripts/publish_next.sh
```

Печатает следующий элемент, ничего не отправляет, ничего не архивирует.

## Пустая очередь

Если в папке нет `*.md` элементов, плановый запуск — чистый no-op (не падает).
Держи очередь наполненной, чтобы канал не «замолчал».
